Arma3 to Pdrive from mikero

WARNING: Arma3P will overwrite arma2's BIN folder. Ensure, this is a separate p drive

This edition let's you choose which drive to use, not just 'P' and will create that drive for you if you let it.

 

Installation

A no nonsense installer is provided.

Do not run this, or any  tool from bis, as administrator.  'Admin' is not what you think it is, it is an alias for 'Microsoft Office Use only'. Consider yourself warned.

Caveats:

  • YOU MUST have COPIES OF
    • ExtractPbo.exe auto installer
    • Derap.exe auto installer
    • depbo dll auto installer
    • deogg.dll auto installer
    • deroad.exe auto installer

    These packages  can be found at dev-heaven

None of this is earth shattering stuff. The only use of this batch file is because you're already using the above tools anyway.
 

Usage

simply click on the cmd file and go make coffee, feed the cat, or hell, talk to your wife.

Implementation

if the drive does not exist, you have the option of letting it either create one, or assign one that you simply forgot

it will create a folder on the c: drive

c:/?drive eg if it doesn't exist

it will create or alter a batch file in C:/?drive/set?drive.bat

for your subsequent use.

?  is the drive letter you chose.

 

----------------

*ALL* relevant rapified files (configs,missions,rvmats, bisurfs,bins) are converted back to humanly readable text.

Your P:\ drive will be set up to successfully use both TB / OB /  buldozer, and create pbo's using pboProject

This batch file currently extracts to the following folders

  • Arma3P removes
  • \A3
  • \dta
  • \bin
  • core
  • and language_xx
     

before extractions begin.

  • No other folders are AFFECTED

Subsequent upgrades to Arma3 will probably alter this list.

Buldozer

note carefully. This batch file removes p:/buldozer.

There is too much crud.dll's and ancilliary ini, cfg and bat files that need to also be on p: to use it.

you are expected to alter your settings in both TB and OB to use ~steam\arma3.exe directly.

your pDrive is a data folder. If you have any exes in there, you are a disaster waiting happen.

bis binarise will crash when making islands because it searches all over your p drive in a desperate hunt for land_xx classes. placing bis tools in there (which bis do if you let them) will confuse the crap out of binarise.
